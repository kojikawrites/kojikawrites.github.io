# Bluesky Post URL Migration

> **Note**: This is a one-time migration tool for pre-existing sites. If you're setting up a new site, this migration has likely already been completed and you won't need to run it.

This directory contains tools for migrating existing Bluesky posts to include human-readable web URLs.

## Files

- **migrate_bluesky_urls.py**: Migration script to add `post_url` field to existing entries in `bluesky_posted.json`
- **python/workflow/post_to_bluesky.py**: Updated to automatically include `post_url` when posting new content to Bluesky

## What Changed

Each entry in `bluesky_posted.json` now includes:
- `post_id`: Local file path (unchanged)
- `post_uri`: ATProto URI like `at://did:plc:xxx/app.bsky.feed.post/rkey` (unchanged)
- `post_cid`: Content identifier (unchanged)
- **`post_url`**: Human-readable Bluesky web URL like `https://bsky.app/profile/did:plc:xxx/post/rkey` (**NEW**)

## How It Works

The `post_url` is generated by:
1. Parsing the DID and rkey from `post_uri`
2. Constructing `post_url`: `https://bsky.app/profile/{DID}/post/{rkey}`

Note: The DID is used directly in the URL rather than resolving it to a handle, as Bluesky accepts both formats.

## Usage

### Migrate Existing Data

```bash
python migrate_bluesky_urls.py src/.sites/hiivelabs.com/state/bluesky_posted.json
```

### New Posts

New posts automatically include `post_url` when created via `python/workflow/post_to_bluesky.py` - no migration needed.

### Example Manual Migration

If automated migration fails, you can manually add `post_url` to each entry:

```json
{
  "post_id": "posts/hiivelabs.com/2025-01-06-cellular-automata.mdx",
  "post_uri": "at://did:plc:uxb4wvzgll6c47yxxpyqlib7/app.bsky.feed.post/3lf4sdvgkez25",
  "post_cid": "bafyreidtb6dvehh5ubw7f7wooa6wxjy5fo4l5miubyvov227j63asrskhu",
  "post_url": "https://bsky.app/profile/did:plc:uxb4wvzgll6c47yxxpyqlib7/post/3lf4sdvgkez25"
}
```

## Technical Details

- Future posts automatically include `post_url` without manual intervention
- The script also removes the deprecated `profile_id` field if present
- No external dependencies required (uses only Python standard library)
